# Appendix B: Glossary for Old ABAPers

> *BTP Terms Translated to What You Know*

---

| BTP Term | What It Means | Old-World Equivalent |
|----------|---------------|---------------------|
| **Global Account** | Your master contract with SAP | The license agreement |
| **Subaccount** | Isolated environment for work | Like a separate SAP system |
| **Destination** | Named connection configuration | SM59 RFC destination |
| **Cloud Connector** | Tunnel to on-premise | Like SAProuter but modern |
| **Service Instance** | Enabled feature | Like an installed add-on |
| **Entitlement** | Permission to use a service | License allocation |
| **Environment** | Runtime platform | Application server |
| **BAS** | Business Application Studio (IDE) | WebIDE / Eclipse |
| **RAP** | RESTful ABAP Programming | New dev model for ABAP |
| **CDS** | Core Data Services | Like DDIC views but modern |
| **OData** | REST-like API protocol | Like RFC but HTTP |
| **Fiori Elements** | Template-based UI | Like ALV but for web |
| **MTA** | Multi-Target Application | Deployment package |
| **Work Zone** | Central launchpad | Like Fiori Launchpad |
| **Clean Core** | No modifications to standard | The goal SAP always wanted |
| **Side-by-side** | Extension outside S/4 | Z-code in BTP not S/4 |
| **Joule** | AI assistant | Nothing comparable (new!) |
| **Skill** | Single AI capability | Like one function module |
| **Agent** | AI orchestrator | Like calling multiple FMs |
| **Grounding** | Teaching AI context | Like custom documentation |
| **Action Project** | API wrapper for Joule | Wrapper function group |
| **RISE** | SAP's cloud bundle | Managed S/4 + services |
| **Private Edition** | Dedicated S/4 instance | Like hosted on-prem |
| **Public Edition** | Shared S/4 (multi-tenant) | New, no direct equivalent |
| **gCTS** | Git-based transport | CTS but with Git |
| **Key User Extensibility** | Low-code customization | Like easy enhancement |
